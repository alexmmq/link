Сервис для коротких ссылок

Используемый стек:
Java 21
JUnit5

Приложение при запуске в цикле заправшивает необходимую информацию в зависимости от состояния переменной isLoggedIn.
В состоянии незарегистрированного пользователя приложение выводит в консоль следующие инструкции:

"Please enter following commands to proceed:
'init' - to create a UUID
'login' - to log in using existing UUID"
'exit' - to exit the program"

1) init - инициализирует создание уникального UUID в памяти приложения, после успешного создания пользователь также 
будет перенаправлен в начальное меню;
После успешного ввода init пользователю предоставляется ввести параметр для возможного количества использования
коротких ссылок созданных под данным UUID - число не может быть отрицательным, при вводе 0 - ссылки могут быть
использованы без ограничений (кроме случая истечения срока давности ссылки, заданной программно). В консоль будут 
выведены следующие инструкции:
"Please enter amount of times a link can be used, number cannot be less 0, in case of 0 - no limit for links"

2) login - выводит в консоль существующие UUID, которые могут быть использованы для входа в систему и последующей работы
с короткими или длинными ссылками. После успешного ввода активного UUID пользователь попадает во внутреннее меню 
использования сервиса коротких ссылок;
3) exit - инициализирует выход из программы.

После успешного введения существующего UUID в login пользователю выводятся в консоль следующие инструкции:
"Please use following commands:
's'  - to use a short link,
'l' - to create a short link from long one"
'e' - for editing links,
'r' - for removing links
'logout' - for logout
'exit' - for exit"

4) s - пользователю выводится список существующих активных ссылок для использования, если таковых нет - пользователь 
будет оповещен об этом и вернется в исходное меню. Для использования короткой ссылки необходимо скопировать активную
короткую ссылку и ввести в терминал, программа автоматически откроет вкладку в браузере по умолчанию.
5) l - пользователю будет выведено следующее сообщение с требованием введения длинной ссылки для генерации короткой,
как показано ниже:
"Enter a link to be transformed into short link"

6) e - пользователю будет выведено следующее сообщение с требованием ввести существующую длинную ссылку для 
редактирования - в случае если пользователь хочет использовать активную короткую ссылку для другой длинной ссылки, как
следует из следующего сообщения:
"Enter a long link to edit"

После успешного введения пользователю будет необходимо ввести новую ссылку ассоциированную с короткой ссылкой, как 
показано ниже:
"Please enter a new value for a long link"

После успешного введения короткая ссылка меняет ассоциацию и готова к использованию, о чем будет сообщено пользователю

7) r - пользователю будет необходимо ввести короткую ссылку для удаления из списка существующих. Для удобства выводится
список всех активных коротких ссылок доступных для удаления.
   "Enter a short link to remove"

В случае успешного введения короткой ссылки пользователь будет уведомлен об удалении короткой ссылки.

8) logout - после введения данной команды пользователь будет возвращен в изначальное меню.
9) exit - инициализирует выход из программы.
